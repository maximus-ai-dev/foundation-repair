Please use a desktop computer

Foundation Repair isn't available on mobile devices. This is a deliberate security decision, not a technical limitation.

When you sign a transaction on a desktop wallet extension like MetaMask, you can see exactly what contract you're calling, what function is being invoked, and what parameters are being sent — all in a dedicated popup that the website cannot tamper with.

Mobile wallet apps open websites inside a built-in browser (a "webview") that the wallet controls. This makes it much harder to independently verify what you're signing, and easier for a compromised or spoofed page to present misleading transaction details. Since this tool directly interacts with your wallet and your NFTs, we don't think that's an acceptable risk.

Open this link on a laptop or desktop computer with a browser extension wallet (MetaMask, Rabby, Coinbase Wallet, etc.) installed.

Foundation Repair

Foundation is shutting down. If you have art stuck in an auction or a buy-now listing, this tool helps you get it back into your wallet — and optionally burn it — without waiting for Foundation to do it for you.

It's free, open-source, and runs entirely in your browser. Nothing leaves your computer except the transactions you sign in your wallet.

Security & risks

This tool sends transactions from your wallet to a smart contract. That's a serious action — you should understand what's actually happening, what can go wrong, and what we've done to reduce the risk. Read this before you sign anything.

What the tool does

It calls three public functions on the verified Foundation Market contract (cancelReserveAuction, cancelBuyPrice) and optionally burn on your NFT contract. That's it. No backend, no accounts, no hidden logic, no data collection. The full source is on GitHub.

Risks you're taking

  1. Human error. You could paste the wrong token ID or burn the wrong piece. The tool shows a confirmation panel and requires a checkbox for burns, but nothing can protect you from signing a transaction for the wrong NFT.
  2. Your wallet or browser could be compromised. If your MetaMask is a fake extension, or your browser has malicious extensions, or your OS is infected — no amount of care in this tool matters. Verify your wallet came from the official source.
  3. This tool may have bugs we missed. The code passed an automated security audit and 64 regression tests, but it has not received a formal human security review. Use it accordingly.
  4. Underlying libraries and contracts might have bugs. We pin ethers.js to a specific version and rely on Foundation's audited contracts. If either has an undisclosed bug, we inherit it.
  5. Your wallet popup is your last line of defense. Before clicking "Confirm" in MetaMask or Rabby, always verify the target contract address and function name match what the tool says. Don't trust the website — trust the wallet.

What we've done to reduce the risk

  • No external code at runtime. ethers.js v6.13.4 is vendored locally and pinned with a cryptographic integrity hash. If anyone tampers with it, your browser refuses to run it.
  • No HTML injection possible. Every piece of text on the page is rendered through textContent, never innerHTML. There is no XSS attack surface.
  • Strict Content Security Policy. Only scripts from this domain can run. No inline scripts, no eval, no CDN dependencies. External connections are limited to HTTPS only.
  • Hardcoded transaction targets. The Foundation contract address and chain ID are constants baked into the code. Nothing a user pastes can redirect a transaction to a different contract.
  • Hardened URL parser. The tool only accepts links from the exact etherscan.io and foundation.app domains — not look-alikes like evil-etherscan.io.
  • Ethereum mainnet only. The tool refuses to send any write transaction on any other chain, so you can't accidentally broadcast a cancel on the wrong network.
  • Desktop only. Mobile wallet browsers make it harder to verify what you're signing, so they're blocked outright.
  • Clickjacking defenses. This page cannot be embedded in an iframe, so no malicious site can overlay invisible buttons on top of yours.
  • Public source + MIT license. Anyone can read, verify, fork, or host their own copy.

Common questions

Is this going to cost me money?

Every action is a regular Ethereum transaction, so you'll pay the network gas fee (usually a few dollars on mainnet). Nothing goes to this tool — the transaction goes directly to the Foundation contract and costs exactly the same as if you did it yourself on Etherscan.

Is it safe?

The tool only calls two public functions on the verified Foundation Market contract (cancelReserveAuction and cancelBuyPrice) and optionally burn on your NFT contract. Everything runs in your browser. You can read the source on GitHub. Every transaction is shown to you in your wallet before you sign it — always double-check what you're signing.

I don't have a wallet. What do I do?

You need the same wallet you originally used to list the piece on Foundation — the contract won't let anyone else cancel your listing. If you used MetaMask on Foundation, install the MetaMask browser extension and sign in with the same seed phrase.

What if my auction has a bid?

It can't be cancelled. The contract only allows the seller to cancel an auction that has received zero bids. If there's a bid and the auction hasn't ended, you'll have to wait for it to conclude.

What does "escrow" mean?

When you list on Foundation, the NFT physically moves from your wallet to the marketplace contract until someone buys it or you cancel. Cancelling moves it back. That's why step 2 says "return NFT" — it's literally coming back to you.

Why would I burn a piece? Isn't that destroying my art?

The art itself isn't the token — the art is the image, audio, or video file the token points to. Burning only destroys the specific token on Foundation's shared contract. The common reason artists do this is to remint the same work on a sovereign contract they control — often deployed through Manifold or as a custom contract. That way the piece isn't dependent on Foundation's infrastructure or anyone else's marketplace to exist on-chain. If you're not planning to remint, there's no reason to burn.
